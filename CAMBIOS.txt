TRLista2.0 — Migración a backend para Firestore (2026-03-13)

1. Persistencia del checklist
- Se eliminó el acceso directo a Firestore desde el navegador.
- Ahora el frontend usa `/api/checklist-get`, `/api/checklist-save` y `/api/checklist-history`.
- Vercel se conecta a Firestore mediante Firebase Admin SDK y variables de entorno seguras.

2. Manejo de errores
- Cuando el servidor no responde, la app intenta usar una copia local del checklist ya leída.
- El badge superior muestra advertencias de sincronización en lugar de fallar silenciosamente.
- El calendario mantiene sus fechas marcadas mediante caché local por documento.

3. Histórico desbloqueable
- Se conserva el botón para desbloquear listas históricas con contraseña desde `estantes!G2`.
- Guardar, limpiar y mover entre listas siguen respetando la fecha histórica seleccionada.

4. Frontend
- Se removieron los scripts cliente de Firebase del `index.html`.
- El calendario local estable se mantiene sin depender de Flatpickr.
